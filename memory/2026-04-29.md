Added `llamacpp` as third LLM provider for Envestero AI persona backtests.

**Model:** Qwen2.5-7B-Instruct-Q3_K_M.gguf (3.6GB, from bartowski/HF)  
**GPU:** AMD Radeon Vega M GH (4GB HBM2) — model fully on VRAM  
**Service:** `llama-inference.service` (systemd, port 11435)

**Key lessons learned:**
1. `llama-cli --list-devices` shows no devices without `render` group — must run as `Group=render` (same as embedding service)
2. Docker containers can't reach `127.0.0.1` on host — use Docker bridge gateway `172.18.0.1`
3. `host-gateway` hostname only works in newer Docker Compose — use explicit IP
4. Firewall blocks Docker→host by default — `sudo ufw allow from 172.18.0.0/16 to any port 11435`
5. llama-server `--temp 0.0` enforces determinism server-side — no per-call temperature needed

**Both llama services fit in 4GB HBM2:**
- llama-embedding (11434): 152MB — nomic-embed for MemoryGraph
- llama-inference (11435): 3,935MB — Qwen2.5-7B for Envestero
- Total: 4,087MB / 4,096MB ✅
