## 🦙 llama.cpp Hot-Swap Infrastructure (2026-04-30)

### What Was Built
PowerShell API server (`start_llama.ps1`) that:
- Dynamically discovers all `.gguf` models in the models folder
- Assigns ports sequentially from 8000
- Exposes REST API on port 9000 for start/stop/list/reload
- Runs as Windows Scheduled Task on kids-01 (auto-starts at boot)
- inv-toan: started manually on demand

### OCP Files
- `/srv/openclaw_projects/Envestero/start_llama.ps1` — hot-swap API server
- `/srv/openclaw_projects/Envestero/install_service.ps1` — Windows Task Scheduler installer
- `/srv/openclaw_projects/Envestero/README.md` — full LLM infrastructure docs

### API Endpoints (port 9000 on each machine)
```
GET  /list                     → available + running models
GET  /start?model=<name>       → start model
POST /stop?model=<name>        → stop model
POST /stopall                  → stop all
POST /reload                   → rescan models directory
```

### llama.cpp Build on Windows — Key Lesson
CUDA 12.4 only supports VS 2017-2022. VS 2026 BuildTools causes:
`fatal error C1189: unsupported Microsoft Visual Studio version`
Fix options:
1. Use `-allow-unsupported-compiler` CUDA flag (risky)
2. Download pre-built CUDA binary from llama.cpp GitHub releases
3. Use winget install (installs Vulkan-only, no CUDA)
→ Best path: download pre-built `llama-bXXXX-bin-win-cuda-12.4-x64.zip` + `cudart-llama-bin-win-cuda-12.4-x64.zip`
