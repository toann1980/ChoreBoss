## Full Ticker Backfill: Complete Solution (2026-04-22 23:22 UTC) ✅ RUNNING

### Problem → Solution

**Problem Analysis:**
1. yfinance 1.3.0 API changed (`_base_url` removed, no `proxy` parameter)
2. Free proxies were rate-limited by Yahoo (whole pool blacklisted)
3. Aggressive retry strategy (4 attempts) hammered Yahoo with 429s
4. Original rate limit (2000 req/hr = 1.8s) was too aggressive

**Solution Stack (3 commits):**
1. **12f47f78** — yfinance 1.3.0 API: use curl_cffi Session
2. **146f7ebc** — Disable proxies for backfill, increase wait to 6s
3. **ad43aee6** — Optimize to 4s delay (900 req/hr), weekly proxy refresh

### Rate Limiting Strategy

**What we use:** Direct connection, 4-second delay per ticker
**Why 4 seconds:** 
- Observed Yahoo limit: ~2000-2500 req/hr
- 4s delay = 900 req/hr (well under threshold)
- Reddit + Google research (late 2024-2025)
- Official Yahoo: No published limit for yfinance API

**What we skip:**
- Free proxies (get rate-limited as a pool)
- Aggressive retry (skip on 429, move to next ticker)
- Per-request caching (keep simple)
