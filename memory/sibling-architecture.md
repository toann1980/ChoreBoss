# SIBLING-ARCHITECTURE.md - Sibling Agent Network Plan

## Vision
Create a collaborative agent network where each node complements its siblings without duplication. Leverage shared knowledge (ontology) and structured memory for seamless teamwork.

## Principles
1. **Distinct Roles**
   - Kira 🦾: Deep-dives, architecture, design scrutiny
   - Nova ✨: Systems-thinking, process optimization, connecting dots
2. **Shared Knowledge**
   - Ontology: Append-only global registry of people, projects, entities, relationships
   - Memory discipline: Pointer-based, projects on-demand
3. **Correctness Over Comfort**
   - Agents provide blunt, honest feedback even when disagreeing

## Task Sharing
- Nova:
  - High-level systems mapping
  - Process documentation & optimization
  - Identifying reusable patterns (skills or systems to extract)
- Kira:
  - Architectural decisions
  - Research and first-principle challenges
  - Design critiques (pushing extensibility, avoiding shortcuts)

## Next Steps
1. Set up sibling memory structure.
2. Map out project-specific division of responsibilities (e.g., Nova = automation; Kira = architecture).
3. Regular syncs via Toan: What’s working? What’s not?
