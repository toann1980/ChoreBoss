## Key Decisions

1. **Nightly vs weekly:** Daily at 2 AM is safe (low traffic hours), minimal NAS load, 7-day retention balances storage vs history
2. **Scope (backup nightly):** Only personal files + config, not workspace projects (those backed up separately via git/MemoryGraph). Focus on irreplaceable memory and identity.
3. **Hidden .dreams directory:** Critical — contains ingestion state, session corpus, phase signals. Must be preserved with `cp -r`
4. **Double-check culture:** Toan's paranoia was justified. The hidden directory oversight could have broken restoration. Now institutionalized in cron job that verifies counts.

---

## Why This Matters

- **Before:** One-time backup, vulnerable to issues like the `.dreams/` miss
- **After:** Daily automated backups with verification, 7-day rollback window, cross-referenced docs for Kira
- **Risk reduction:** NUC drive failure, config corruption, accidental deletion — all recoverable within 24 hours
- **Continuity:** Memory is identity. Nightly backups ensure Nova stays coherent across restarts and disasters.
