# Alembic Migration Guide

Quick reference for working with database migrations.

---

## Setup (First Time)

```bash
# Already done! But if you need to reinit:
alembic init migrations

# Edit alembic.ini:
# sqlalchemy.url = sqlite:///choreboss.db

# Edit migrations/env.py:
# from choreboss.models import Base
# target_metadata = Base.metadata
```

---

## Common Commands

### View Status
```bash
# Current revision
alembic current

# All revisions
alembic history

# Latest revision
alembic heads
```

### Apply Migrations
```bash
# Apply all pending migrations
alembic upgrade head

# Apply next migration
alembic upgrade +1

# Apply specific revision
alembic upgrade b17de874045a
```

### Revert Migrations
```bash
# Revert to previous revision
alembic downgrade -1

# Revert all tables
alembic downgrade base

# Revert to specific revision
alembic downgrade b17de874045a
```

### Generate Migrations

#### Auto-Detect Model Changes
```bash
# Generate migration from model changes
alembic revision --autogenerate -m "Add user roles"

# Preview without creating
alembic revision --autogenerate -m "Test" --sql
```

#### Manual Migration
```bash
# Create empty migration (you fill in the SQL)
alembic revision -m "Manual: custom SQL"
```

## Workflow Example

### 1. Modify a Model
```python
# choreboss/models/chore.py
class Chore(Base):
    __tablename__ = "chores"
    # ... existing fields ...
    priority = Column(Integer, default=1)  # NEW FIELD
```

### 2. Generate Migration
```bash
alembic revision --autogenerate -m "Add priority field to chores"

# Creates: migrations/versions/abc12345_add_priority_field_to_chores.py
```

### 3. Review Migration
```python
# migrations/versions/abc12345_add_priority_field_to_chores.py
def upgrade() -> None:
    op.add_column('chores', sa.Column('priority', sa.Integer(), default=1))

def downgrade() -> None:
    op.drop_column('chores', 'priority')
```

### 4. Test Migration
```bash
# Apply
alembic upgrade head

# Verify it worked
sqlite3 choreboss.db ".schema chores"

# Test downgrade
alembic downgrade -1

# Test upgrade again
alembic upgrade head
```

### 5. Commit to Git
```bash
git add migrations/versions/abc12345_add_priority_field_to_chores.py
git commit -m "Add priority field to chores"
```

## Advanced Usage

### Branching (Multiple Heads)
```bash
# Create migration from non-head revision (creates branch)
alembic revision --autogenerate -m "Branch change" --rev-id branch1 \
  --depends-on b17de874045a

# Merge branches
alembic revision -m "Merge branches" --branch-label merged
```

### Custom Operations
```python
# migrations/versions/xxx_custom_migration.py
def upgrade() -> None:
    # Raw SQL
    op.execute("INSERT INTO chores (name) VALUES ('Default Chore')")
    
    # Column operations
    op.add_column('chores', sa.Column('active', sa.Boolean(), default=True))
    op.alter_column('chores', 'description', new_column_name='details')
    
    # Constraints
    op.create_index('idx_chores_person', 'chores', ['person_id'])
    op.create_unique_constraint('uq_chores_name', 'chores', ['name'])

def downgrade() -> None:
    op.drop_index('idx_chores_person')
    op.drop_constraint('uq_chores_name')
    op.drop_column('chores', 'active')
```

### Environment Variables
```bash
# Use DATABASE_URL environment variable (overrides alembic.ini)
export DATABASE_URL="postgresql://user:pass@localhost/choreboss"
alembic upgrade head

# CI/CD
alembic upgrade head  # Uses DATABASE_URL from environment
```

## Migration File Format

### Minimal Migration
```python
from alembic import op
import sqlalchemy as sa

revision = 'abc12345'
down_revision = 'b17de874045a'
branch_labels = None
depends_on = None

def upgrade() -> None:
    # Code to apply migration
    pass

def downgrade() -> None:
    # Code to revert migration
    pass
```

### Typical Autogenerated Migration
```python
def upgrade() -> None:
    op.create_table(
        'new_table',
        sa.Column('id', sa.Integer(), nullable=False),
        sa.Column('name', sa.String(length=50), nullable=False),
        sa.PrimaryKeyConstraint('id'),
    )

def downgrade() -> None:
    op.drop_table('new_table')
```

## Common Operations

### Add Column
```python
op.add_column('table_name', 
    sa.Column('new_column', sa.String(100), nullable=True))
```

### Drop Column
```python
op.drop_column('table_name', 'old_column')
```

### Rename Column
```python
op.alter_column('table_name', 'old_name', new_column_name='new_name')
```

### Add Index
```python
op.create_index('idx_table_column', 'table_name', ['column_name'])
```

### Add Constraint
```python
op.create_unique_constraint('uq_table_column', 'table_name', ['column_name'])
```

### Drop Constraint
```python
op.drop_constraint('uq_table_column', 'table_name')
```

### Rename Table
```python
op.rename_table('old_table', 'new_table')
```

### Execute Raw SQL
```python
op.execute("UPDATE chores SET active = 1 WHERE person_id IS NOT NULL")
```

## Troubleshooting

### Check Syntax
```bash
# Verify migration without running it
alembic upgrade head --sql
```

### Manual Migration Verification
```bash
# View current database schema
sqlite3 choreboss.db ".schema"

# Check what Alembic thinks is current
alembic current
```

### Restore from Commit
```bash
# If migration is broken, revert git commit
git revert abc12345

# Re-downgrade database
alembic downgrade -1

# Try again
git pull
alembic upgrade head
```

### View Migration History
```bash
alembic history -v  # Verbose
alembic history --sql  # Show SQL
```

## Tips & Tricks

### Template Variables in Migrations
```python
def upgrade() -> None:
    # Alembic provides these automatically:
    # op - operations object
    # sa - sqlalchemy module
    # config - alembic config
    
    # Get config values:
    # config.get_section('alembic')
    pass
```

### Conditional Migrations
```python
from sqlalchemy import inspect
from alembic import op

def upgrade() -> None:
    # Only add column if it doesn't exist
    # (needed for multiple branches)
    inspector = inspect(op.get_bind())
    columns = [col['name'] for col in inspector.get_columns('chores')]
    
    if 'priority' not in columns:
        op.add_column('chores', sa.Column('priority', sa.Integer()))
```

### Bulk Data Operations
```python
def upgrade() -> None:
    # Insert seed data
    op.execute("""
        INSERT INTO chores (name, description) 
        VALUES ('Default', 'Default chore')
    """)

def downgrade() -> None:
    # Clean up
    op.execute("DELETE FROM chores WHERE name = 'Default'")
```

## Database Schema Version Tracking

Alembic tracks applied migrations in the `alembic_version` table:

```sql
-- View applied migrations
SELECT * FROM alembic_version;

-- Output:
-- version_num
-- b17de874045a

-- Manual reset (emergency only):
-- DELETE FROM alembic_version;
-- alembic stamp head  # Reset to current
```

## Pre-Deployment Checklist

- [ ] Generate migration: `alembic revision --autogenerate -m "..."`
- [ ] Review migration file (no data loss)
- [ ] Test upgrade: `alembic upgrade head`
- [ ] Test downgrade: `alembic downgrade -1`
- [ ] Test upgrade again: `alembic upgrade head`
- [ ] Commit to git: `git add migrations/versions/...`
- [ ] Deploy (runs `alembic upgrade head` in docker/CI)
