## How to Use

### Create New Migration (Auto-detect)
```bash
alembic revision --autogenerate -m "Add column X to table Y"
```

### Create Manual Migration
```bash
alembic revision -m "Manual: describe change"
# Edit migrations/versions/xxxx_describe_change.py
```

### Apply All Migrations
```bash
alembic upgrade head
```

### Apply Specific Revision
```bash
alembic upgrade +1              # Next migration
alembic upgrade b17de874045a    # Specific revision
```

### Revert Last Migration
```bash
alembic downgrade -1            # Previous migration
alembic downgrade base          # Drop all tables
```

### View Current State
```bash
alembic current                 # Current revision
alembic history                 # All migrations
alembic heads                   # Latest revisions
alembic branches                # Branch points
```

## Migration Workflow

### Development
1. Create/modify models in `choreboss/models/`
2. Generate migration: `alembic revision --autogenerate -m "Description"`
3. Review migration file in `migrations/versions/`
4. Test: `alembic upgrade head` → `alembic downgrade -1` → `alembic upgrade head`
5. Commit migration to git

### Production Deployment
1. Pull latest migrations from git
2. Run: `alembic upgrade head`
3. Database schema updated automatically
4. **Never edit migrations after commit**

### Rollback (Emergency)
```bash
alembic downgrade <previous-revision-id>
```

## Best Practices

### ✅ Do
- **Auto-generate:** Use `--autogenerate` for model changes
- **Review:** Always review generated migrations before committing
- **Test:** Test both upgrade and downgrade locally
- **Commit:** Commit migration files to git with models
- **Descriptive messages:** `alembic revision --autogenerate -m "Add user roles table"`
- **Small changes:** One logical change per migration

### ❌ Don't
- **Edit committed migrations:** Never change migration files after commit
- **Manual SQL:** Avoid SQL directly (use operations API)
- **Data loss:** Don't drop columns without backup plan
- **Rename tables:** Use create + copy + drop pattern
- **Skip downgrade:** Always implement downgrade() function
- **Production only:** Test migrations in dev/staging first

## Troubleshooting

### Migration Not Detected
```bash
# Check if target_metadata is set in env.py
# Verify models are imported in env.py
# Check sqlalchemy.url is correct
alembic revision --autogenerate -m "Test" --sql  # Preview without creating
```

### Can't Downgrade
```bash
# Make sure downgrade() function is properly implemented
# Check foreign key constraints (may need CASCADE)
alembic downgrade -1 --sql  # Preview downgrade
```

### Lost Tracking
```bash
# Alembic versions table tracks applied migrations
# Never delete from alembic_version table manually
sqlite3 choreboss.db "SELECT * FROM alembic_version;"
```
