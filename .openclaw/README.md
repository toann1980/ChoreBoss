# Project Local Notes Convention

Use a `.openclaw/` folder in each project for local-only documentation, decisions, and operational notes.

Recommended contents:
- setup and deployment notes
- troubleshooting writeups
- architecture decisions
- project-specific runbooks
- temporary working notes that should stay local

Rules:
- Check `.openclaw/` first when you need repo-specific context.
- Keep project knowledge in one place instead of scattering it through the repo root.
- Prefer concise notes that are easy to scan later.
